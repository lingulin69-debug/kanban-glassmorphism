## 1. 檔案寫入問題

### 問題 1.1：`create_file` 工具內容三重複製
- **現象**：使用 AI 工具的 `create_file` 寫入大檔案時，內容被重複三次貼入
- **原因**：工具在處理大量內容時的 bug
- **解決方案**：
  - 對已存在的檔案，改用 `replace_string_in_file` 做局部編輯
  - 需要整檔重寫時，建立 Node.js CJS 腳本 (`write-app.cjs`) 用 `fs.writeFileSync` 寫入
  - **永遠不要**用 `create_file` 覆寫已存在的大檔案

### 問題 1.2：PowerShell `Set-Content` 中文亂碼
- **現象**：用 PowerShell 的 `Set-Content` 寫入含中文的 JSX 檔案，中文字元被替換成亂碼
- **原因**：PowerShell 5.1 預設使用 Windows-1252 編碼，無法正確處理 UTF-8 中文
- **解決方案**：
  - 在 Node.js 腳本中將中文字元轉為 `\uXXXX` Unicode 跳脫序列
  - 或使用 `[System.IO.File]::WriteAllText($path, $content, [System.Text.Encoding]::UTF8)` 
  - **最佳做法**：避免用 PowerShell 寫入含非 ASCII 字元的檔案

### 問題 1.3：PSReadLine 緩衝區溢出
- **現象**：透過終端執行過長的 PowerShell 命令時，PSReadLine 報錯或截斷內容
- **原因**：PSReadLine 輸入緩衝區有長度限制（~8KB）
- **解決方案**：
  - 將大量內容寫入臨時 `.cjs` 腳本，用 `node script.cjs` 執行
  - 避免在終端命令中內嵌大段程式碼

## 2. Vite 開發環境

### 問題 2.1：Tailwind CSS v4 設定方式改變
- **現象**：使用 `tailwind.config.js` 設定無效
- **原因**：Tailwind CSS v4 改為 CSS-first 設定，不再使用 JS 設定檔
- **解決方案**：
  ```css
  /* src/index.css */
  @import "tailwindcss";
  ```
  ```js
  // vite.config.js
  import tailwindcss from "@tailwindcss/vite"
  export default defineConfig({
    plugins: [react(), tailwindcss()]
  })
  ```
  - 安裝 `@tailwindcss/vite` 而非 `tailwindcss` + `postcss` + `autoprefixer`

### 問題 2.2：HMR 熱更新偶爾失效
- **現象**：修改檔案後瀏覽器不更新
- **解決方案**：確保每個元件檔案只有一個 `export default`，且檔名與元件名一致

## 3. Glassmorphism（玻璃擬態）效果

### 問題 3.1：玻璃效果不可見
- **現象**：`backdrop-blur` + 半透明背景的卡片看起來完全透明或白色
- **原因**：背景沒有可模糊的元素（純色背景）
- **解決方案**：
  - 在背景添加裝飾性模糊色塊（blobs）：
    ```jsx
    <div className="pointer-events-none fixed inset-0 z-0">
      <div className="absolute -top-32 -left-32 h-[500px] w-[500px] rounded-full bg-neutral-300/25 blur-3xl" />
    </div>
    ```
  - 卡片加上 `backdrop-saturate-[180%]` 增加飽和度
  - Safari 需要 `-webkit-backdrop-filter` 前綴：
    ```jsx
    style={{ WebkitBackdropFilter: 'blur(16px) saturate(180%)' }}
    ```

### 問題 3.2：Modal 動畫卡頓
- **現象**：彈窗開啟/關閉時明顯延遲
- **原因**：overlay 使用 `motion.div` 觸發重排
- **解決方案**：
  - 背景遮罩改用普通 `<div>` 而非 `<motion.div>`
  - 動畫元素加上 `willChange: 'transform, opacity'`
  - 使用 `spring` 物理動畫而非 `tween`（siffness: 300, damping: 30）

## 4. Supabase 雲端同步

### 問題 4.1：資料庫表格需手動建立
- **步驟**：
  1. 在 Supabase Dashboard → SQL Editor 執行建表 SQL
  2. 啟用 Row Level Security (RLS)
  3. 設定 Realtime 監聽
- **建表 SQL 範本**：
  ```sql
  CREATE TABLE IF NOT EXISTS tasks (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    title TEXT NOT NULL DEFAULT '',
    description TEXT DEFAULT '',
    column_name TEXT DEFAULT 'todo',
    priority TEXT DEFAULT 'medium',
    start_date TEXT,
    end_date TEXT,
    tags JSONB DEFAULT '[]'::JSONB,
    color TEXT DEFAULT '#171717',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
  );
  ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "Allow all" ON tasks FOR ALL USING (true) WITH CHECK (true);
  ALTER PUBLICATION supabase_realtime ADD TABLE tasks;
  ```

### 問題 4.2：camelCase ↔ snake_case 轉換
- **現象**：React 用 `startDate`，Supabase 用 `start_date`
- **解決方案**：建立 `toDb()` / `fromDb()` 轉換函式
  ```js
  const toDb = (t) => ({ ...t, column_name: t.column, start_date: t.startDate, end_date: t.endDate })
  const fromDb = (r) => ({ ...r, column: r.column_name, startDate: r.start_date, endDate: r.end_date })
  ```

### 問題 4.3：Realtime 訂閱記憶體洩漏
- **解決方案**：在 `useEffect` 的 cleanup 中取消訂閱
  ```js
  useEffect(() => {
    const channel = supabase.channel('tasks-changes')
      .on('postgres_changes', { event: '*', schema: 'public', table: 'tasks' }, handler)
      .subscribe()
    return () => { supabase.removeChannel(channel) }
  }, [])
  ```

## 5. GitHub + Vercel 部署

### 問題 5.1：Vercel 自動部署設定
- **步驟**：
  1. `git init` → `git remote add origin <url>` → `git push`
  2. Vercel CLI: `npx vercel --prod`
  3. 之後每次 `git push` 到 master 自動觸發部署
- **注意**：`.gitignore` 需排除 `node_modules/`、`dist/`、`.env`

### 問題 5.2：環境變數安全
- **重要**：Supabase anon key 可以暴露在前端（受 RLS 保護），但 `service_role` key **絕對不能**放前端
- **做法**：在 Vercel Dashboard → Settings → Environment Variables 設定敏感變數

### 問題 5.3：Vercel 部署後看不到更新（瀏覽器快取）
- **現象**：`git push` 後 Vercel 顯示部署成功，但打開網站仍是舊版
- **原因**：瀏覽器強快取（Service Worker / 200 from disk cache）
- **解決方案**：
  1. `Ctrl + Shift + R`（強制重新整理）
  2. 開無痕視窗測試
  3. 若仍無效，用 `npx vercel --prod` 強制重新部署
  4. 確認 Vercel Dashboard 上最新部署的 Status 為 "Ready"
- **預防**：在 `vercel.json` 加入快取控制：
  ```json
  {
    "headers": [
      { "source": "/(.*)", "headers": [{ "key": "Cache-Control", "value": "public, max-age=0, must-revalidate" }] }
    ]
  }
  ```

### 問題 5.4：Git Push 觸發 Vercel 但未部署到 Production
- **現象**：Vercel 有收到部署但 production URL 未更新
- **原因**：Vercel 可能將 push 部署到 Preview 而非 Production（非 main/master 分支）
- **解決方案**：
  1. 確認推送的分支是 Vercel 設定中的 Production Branch（通常是 `main` 或 `master`）
  2. 或手動執行 `npx vercel --prod` 強制部署到 production

## 6. 樣式與主題切換

### 問題 6.1：亮/暗主題實作策略
- **做法**：用 `useState` + `localStorage` 儲存主題偏好
  ```js
  const [dark, setDark] = useState(() => {
    try { return localStorage.getItem('theme-dark') === 'true' } catch { return false }
  })
  useEffect(() => { localStorage.setItem('theme-dark', dark) }, [dark])
  ```
- 建立 `theme` 物件統一管理色票：
  ```js
  const theme = dark
    ? { bg: "#292828", text: "text-neutral-100", btnBg: "bg-[#F05917]", ... }
    : { bg: "#DED5CC", text: "text-neutral-900", btnBg: "bg-[#F05917]", ... }
  ```
- 子元件透過 `props` 傳遞 `dark` 和 `theme`

### 問題 6.2：Tailwind 動態 class 不生效
- **現象**：`hover:${theme.text}` 這類動態組合的 class 有時不會生效
- **原因**：Tailwind v4 需要在編譯時看到完整 class 名稱
- **解決方案**：
  - 盡量使用完整的條件式而非字串插值
  - 對於 `style` 屬性的色值用原始 hex，不用 Tailwind class
  - 確保所有可能的 class 名稱至少出現一次在原始碼中

## 8. React Hooks 順序與 Temporal Dead Zone

### 問題 8.1：useState 宣告位置導致白屏
- **現象**：頁面完全空白，Console 報 `ReferenceError: Cannot access 'filterLabel' before initialization`
- **原因**：`const [filterLabel, setFilterLabel] = useState(null)` 放在元件中段（line 204），但 `filtered` useMemo（line 171）已引用 `filterLabel`。`const` / `let` 存在 **Temporal Dead Zone (TDZ)**，在宣告語句執行前使用會拋出 ReferenceError
- **關鍵**：`npx vite build` 仍會成功，因為 bundler 只做靜態分析，不執行；此錯誤只在 runtime 才會觸發
- **解決方案**：將 `useState` 宣告移到所有使用該 state 的 `useMemo` / `useEffect` / `useCallback` 之前
- **預防規則**：
  1. **所有 `useState` 宣告集中放在元件函數開頭**，在任何衍生邏輯（useMemo / useEffect / useCallback）之前
  2. 新增 state 時，永遠加在元件頂部的 state 區塊，**不要插在 handler 函式之間**
  3. 修改後先在 dev server 確認頁面能渲染，再 build 部署

### 問題 8.2：音效 `new Audio()` 模組頂層安全性
- **結論**：在模組頂層 `const sound = new Audio('/audio/xxx.wav')` 是安全的（瀏覽器不會阻擋載入，只阻擋自動播放）
- **必須**：所有 `.play()` 呼叫都要加 `.catch(() => {})` 處理自動播放限制
- **必須**：播放前設定 `currentTime = 0` 確保可重複觸發

### 問題 8.3：音量超過 100% 需要 Web Audio API
- **現象**：`Audio.volume` 最大值為 1.0，無法放大音量超過原始檔案音量
- **解決方案**：使用 `AudioContext` + `GainNode`，`gainNode.gain.value = 3.0` 即 300%
- **注意**：播放前需呼叫 `audioCtx.resume()` 解除 suspended 狀態

## 9. 甘特圖拖拽性能優化

### 問題 9.1：拖拽時 UI 卡頓
- **原因**：`onMouseMove` 中直接更新全局 React state 導致每次滑鼠移動都重渲染

- **Ghost 預覽框**：`border: 1px solid rgba(255,255,255,0.2)` + `boxShadow` 微光暈
- **Snap-to-Grid**：`Math.round(dx / dayWidth)` 自動對齊日期邊界

## 11. 給下一位 AI 開發者的參考

### 架構
- 單檔架構 `src/App.jsx`（~800 行），Supabase 即時同步 + localStorage 雙備份
- Glassmorphism 風格：`backdrop-filter: blur()` + 半透明背景 + 細邊框

### 效能
- 移除不必要的 `backdrop-blur` 可大幅提升效能
- Framer Motion 用 `layout="position"` 減少計算
- 甘特圖拖拽務必用 RAF + useRef，禁止 onMouseMove 中 setState

### 常見陷阱
1. **TDZ 白屏**：useState 必須在 useMemo/useEffect 之前宣告
2. **Tailwind v4 動態 class**：`hover:${variable}` 可能不生效，改用 style
3. **PowerShell 中文**：不要用 PowerShell 寫入含中文的檔案
4. **Vercel 快取**：`Ctrl+Shift+R` 或無痕模式測試
5. **音效**：`.play()` 必須加 `.catch()`，音量 > 100% 用 Web Audio API
6. **README.MD 為最高原則**：開發標準以此為準

## 12. Unicode 跳脫序列顯示為亂碼

### 問題 12.1：`\uXXXX` 以原始文字顯示在畫面上
- **現象**：UI 上出現 `\u770B\u677F` 而非「看板」等中文字
- **原因**：CJS 寫入腳本中使用雙反斜線 `\\uXXXX`，導致 JS 字面字串為 `\u770B`（6 個字元），而非 Unicode 字元「看」
- **解決方案**：建立 `fix-unicode.cjs` 腳本批次替換：
  ```js
  content = content.replace(/\\u([0-9A-Fa-f]{4})/g, (_, hex) =>
    String.fromCharCode(parseInt(hex, 16))
  )
  ```
- **預防規則**：
  1. CJS 寫入腳本中直接使用 UTF-8 中文字元，配合 `fs.writeFileSync(path, content, 'utf8')`
  2. 若必須用跳脫序列，使用**單反斜線** `\uXXXX`（JS 會在字串建立時解析為 Unicode 字元）
  3. **絕對不要**使用 `\\uXXXX`（雙反斜線），這會寫入 literal 反斜線字元

## 13. 表單欄位無障礙警告

### 問題 13.1：瀏覽器 autofill 警告 missing id/name
- **現象**：Console 出現 `Input elements should have autocomplete attributes`
- **原因**：`<input>` 和 `<textarea>` 缺少 `id` 和 `name` 屬性，瀏覽器 autofill 機制無法識別
- **解決方案**：為所有表單欄位加上唯一的 `id` + `name` + `autoComplete="off"`
  ```jsx
  <input id="task-title" name="task-title" autoComplete="off" ... />
  ```

## 14. 頁面切換效能優化（看板/月曆/甘特圖卡頓）

### 問題 14.1：framer-motion 進場動畫導致切換卡頓
- **現象**：點擊看板/月曆/甘特圖 tab 後，頁面內容有明顯延遲才出現
- **根因分析**：
  1. `AnimatePresence mode="popLayout"` 強制舊組件離場時重新計算 layout
  2. 所有甘特條同時執行 `scaleX: 0 → 1` spring 動畫（N 個元素 × spring 計算 = 大量 JS）
  3. 看板卡片 `layout="position"` 使 framer-motion 在每次 rerender 時呼叫 `getBoundingClientRect`
  4. `whileHover={{ y: -2 }}` 使 framer-motion 在掛載時對每張卡片設定 hover listener & 測量

### 解決方案：
1. **移除 `AnimatePresence` 包裹頁面切換** — 改用 CSS `@keyframes` 彈跳動畫
2. **CSS GPU 加速彈跳效果**取代 JS 動畫：
   ```css
   @keyframes view-bounce-in {
     0% { opacity: 0; transform: scale(0.96); }
     50% { opacity: 1; transform: scale(1.015); }
     100% { opacity: 1; transform: scale(1); }
   }
   .view-bounce-in {
     animation: view-bounce-in 0.25s cubic-bezier(0.34, 1.56, 0.64, 1) both;
     will-change: transform, opacity;
   }
   ```
   - `transform` 和 `opacity` 在 compositor thread 執行，不觸發 layout 或 paint
3. **移除甘特條 `scaleX` 進場動畫** — 改為 `initial={false}`，直接顯示
4. **移除看板卡片 `layout="position"`** — 改為只保留 `layoutId` 供拖拽
5. **hover 效果從 `whileHover` 改為 CSS** — `hover:-translate-y-0.5 transition-transform`

### 問題 14.2：React 元件不必要的重繪
- **現象**：任一任務更新時，所有看板卡片和甘特列都重新渲染
- **解決方案**：
  1. 抽取 `BoardCard` 和 `GanttRow` 為 `React.memo()` 獨立元件
  2. 用 `useCallback` 包裝 `openEdit`、`handleDrop`、`handleUpdateDates`、`handleBarDrag`、`handleBarClick`
  3. `setTasks` 改為函式更新 `setTasks(prev => prev.map(...))` 避免閉包陳舊值
  4. `content-visibility: auto` CSS 用於 `.gantt-row`，瀏覽器跳過畫面外列的渲染

### 效能規則：
- **framer-motion 動畫只用在「使用者直接互動」**（拖拽、開關 modal）
- **頁面級切換用 CSS `@keyframes`**（GPU 加速，零 JS 成本）
- **大量重複元素（卡片、甘特列）必須 `React.memo`**
- **傳給 memo 元件的 handler 必須 `useCallback`**
- **避免 `layout` prop**，除非真的需要 FLIP 動畫（如拖拽排序）
