## 1. 檔案寫入問題

### 問題 1.1：`create_file` 工具內容三重複製
- **現象**：使用 AI 工具的 `create_file` 寫入大檔案時，內容被重複三次貼入
- **原因**：工具在處理大量內容時的 bug
- **解決方案**：
  - 對已存在的檔案，改用 `replace_string_in_file` 做局部編輯
  - 需要整檔重寫時，建立 Node.js CJS 腳本 (`write-app.cjs`) 用 `fs.writeFileSync` 寫入
  - **永遠不要**用 `create_file` 覆寫已存在的大檔案

### 問題 1.2：PowerShell `Set-Content` 中文亂碼
- **現象**：用 PowerShell 的 `Set-Content` 寫入含中文的 JSX 檔案，中文字元被替換成亂碼
- **原因**：PowerShell 5.1 預設使用 Windows-1252 編碼，無法正確處理 UTF-8 中文
- **解決方案**：
  - 在 Node.js 腳本中將中文字元轉為 `\uXXXX` Unicode 跳脫序列
  - 或使用 `[System.IO.File]::WriteAllText($path, $content, [System.Text.Encoding]::UTF8)` 
  - **最佳做法**：避免用 PowerShell 寫入含非 ASCII 字元的檔案

### 問題 1.3：PSReadLine 緩衝區溢出
- **現象**：透過終端執行過長的 PowerShell 命令時，PSReadLine 報錯或截斷內容
- **原因**：PSReadLine 輸入緩衝區有長度限制（~8KB）
- **解決方案**：
  - 將大量內容寫入臨時 `.cjs` 腳本，用 `node script.cjs` 執行
  - 避免在終端命令中內嵌大段程式碼

## 2. Vite 開發環境

### 問題 2.1：Tailwind CSS v4 設定方式改變
- **現象**：使用 `tailwind.config.js` 設定無效
- **原因**：Tailwind CSS v4 改為 CSS-first 設定，不再使用 JS 設定檔
- **解決方案**：
  ```css
  /* src/index.css */
  @import "tailwindcss";
  ```
  ```js
  // vite.config.js
  import tailwindcss from "@tailwindcss/vite"
  export default defineConfig({
    plugins: [react(), tailwindcss()]
  })
  ```
  - 安裝 `@tailwindcss/vite` 而非 `tailwindcss` + `postcss` + `autoprefixer`

### 問題 2.2：HMR 熱更新偶爾失效
- **現象**：修改檔案後瀏覽器不更新
- **解決方案**：確保每個元件檔案只有一個 `export default`，且檔名與元件名一致

## 3. Glassmorphism（玻璃擬態）效果

### 問題 3.1：玻璃效果不可見
- **現象**：`backdrop-blur` + 半透明背景的卡片看起來完全透明或白色
- **原因**：背景沒有可模糊的元素（純色背景）
- **解決方案**：
  - 在背景添加裝飾性模糊色塊（blobs）：
    ```jsx
    <div className="pointer-events-none fixed inset-0 z-0">
      <div className="absolute -top-32 -left-32 h-[500px] w-[500px] rounded-full bg-neutral-300/25 blur-3xl" />
    </div>
    ```
  - 卡片加上 `backdrop-saturate-[180%]` 增加飽和度
  - Safari 需要 `-webkit-backdrop-filter` 前綴：
    ```jsx
    style={{ WebkitBackdropFilter: 'blur(16px) saturate(180%)' }}
    ```

### 問題 3.2：Modal 動畫卡頓
- **現象**：彈窗開啟/關閉時明顯延遲
- **原因**：overlay 使用 `motion.div` 觸發重排
- **解決方案**：
  - 背景遮罩改用普通 `<div>` 而非 `<motion.div>`
  - 動畫元素加上 `willChange: 'transform, opacity'`
  - 使用 `spring` 物理動畫而非 `tween`（siffness: 300, damping: 30）

## 4. Supabase 雲端同步

### 問題 4.1：資料庫表格需手動建立
- **步驟**：
  1. 在 Supabase Dashboard → SQL Editor 執行建表 SQL
  2. 啟用 Row Level Security (RLS)
  3. 設定 Realtime 監聽
- **建表 SQL 範本**：
  ```sql
  CREATE TABLE IF NOT EXISTS tasks (
    id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
    title TEXT NOT NULL DEFAULT '',
    description TEXT DEFAULT '',
    column_name TEXT DEFAULT 'todo',
    priority TEXT DEFAULT 'medium',
    start_date TEXT,
    end_date TEXT,
    tags JSONB DEFAULT '[]'::JSONB,
    color TEXT DEFAULT '#171717',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
  );
  ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "Allow all" ON tasks FOR ALL USING (true) WITH CHECK (true);
  ALTER PUBLICATION supabase_realtime ADD TABLE tasks;
  ```

### 問題 4.2：camelCase ↔ snake_case 轉換
- **現象**：React 用 `startDate`，Supabase 用 `start_date`
- **解決方案**：建立 `toDb()` / `fromDb()` 轉換函式
  ```js
  const toDb = (t) => ({ ...t, column_name: t.column, start_date: t.startDate, end_date: t.endDate })
  const fromDb = (r) => ({ ...r, column: r.column_name, startDate: r.start_date, endDate: r.end_date })
  ```

### 問題 4.3：Realtime 訂閱記憶體洩漏
- **解決方案**：在 `useEffect` 的 cleanup 中取消訂閱
  ```js
  useEffect(() => {
    const channel = supabase.channel('tasks-changes')
      .on('postgres_changes', { event: '*', schema: 'public', table: 'tasks' }, handler)
      .subscribe()
    return () => { supabase.removeChannel(channel) }
  }, [])
  ```

## 5. GitHub + Vercel 部署

### 問題 5.1：Vercel 自動部署設定
- **步驟**：
  1. `git init` → `git remote add origin <url>` → `git push`
  2. Vercel CLI: `npx vercel --prod`
  3. 之後每次 `git push` 到 master 自動觸發部署
- **注意**：`.gitignore` 需排除 `node_modules/`、`dist/`、`.env`

### 問題 5.2：環境變數安全
- **重要**：Supabase anon key 可以暴露在前端（受 RLS 保護），但 `service_role` key **絕對不能**放前端
- **做法**：在 Vercel Dashboard → Settings → Environment Variables 設定敏感變數

### 問題 5.3：Vercel 部署後看不到更新（瀏覽器快取）
- **現象**：`git push` 後 Vercel 顯示部署成功，但打開網站仍是舊版
- **原因**：瀏覽器強快取（Service Worker / 200 from disk cache）
- **解決方案**：
  1. `Ctrl + Shift + R`（強制重新整理）
  2. 開無痕視窗測試
  3. 若仍無效，用 `npx vercel --prod` 強制重新部署
  4. 確認 Vercel Dashboard 上最新部署的 Status 為 "Ready"
- **預防**：在 `vercel.json` 加入快取控制：
  ```json
  {
    "headers": [
      { "source": "/(.*)", "headers": [{ "key": "Cache-Control", "value": "public, max-age=0, must-revalidate" }] }
    ]
  }
  ```

### 問題 5.4：Git Push 觸發 Vercel 但未部署到 Production
- **現象**：Vercel 有收到部署但 production URL 未更新
- **原因**：Vercel 可能將 push 部署到 Preview 而非 Production（非 main/master 分支）
- **解決方案**：
  1. 確認推送的分支是 Vercel 設定中的 Production Branch（通常是 `main` 或 `master`）
  2. 或手動執行 `npx vercel --prod` 強制部署到 production

## 6. 樣式與主題切換

### 問題 6.1：亮/暗主題實作策略
- **做法**：用 `useState` + `localStorage` 儲存主題偏好
  ```js
  const [dark, setDark] = useState(() => {
    try { return localStorage.getItem('theme-dark') === 'true' } catch { return false }
  })
  useEffect(() => { localStorage.setItem('theme-dark', dark) }, [dark])
  ```
- 建立 `theme` 物件統一管理色票：
  ```js
  const theme = dark
    ? { bg: "#292828", text: "text-neutral-100", btnBg: "bg-[#F05917]", ... }
    : { bg: "#DED5CC", text: "text-neutral-900", btnBg: "bg-[#F05917]", ... }
  ```
- 子元件透過 `props` 傳遞 `dark` 和 `theme`

### 問題 6.2：Tailwind 動態 class 不生效
- **現象**：`hover:${theme.text}` 這類動態組合的 class 有時不會生效
- **原因**：Tailwind v4 需要在編譯時看到完整 class 名稱
- **解決方案**：
  - 盡量使用完整的條件式而非字串插值
  - 對於 `style` 屬性的色值用原始 hex，不用 Tailwind class
  - 確保所有可能的 class 名稱至少出現一次在原始碼中

## 7. 跨瀏覽器相容性

### 問題 7.1：Safari backdrop-filter
- 需要 `-webkit-` 前綴
- React 中設定方式：`style={{ WebkitBackdropFilter: 'blur(16px)' }}`

### 問題 7.2：拖放 (Drag & Drop) API
- `onDragOver` 必須呼叫 `e.preventDefault()` 才能觸發 `onDrop`
- 拖放時的視覺反饋用 `classList.add/remove` 即時控制
